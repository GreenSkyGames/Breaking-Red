The Riding Hood Readme File

Description: In the CharacterSelection scene the user picks Riding Hood's color.

Components:
  1. Sprite Renderer
    On layer level 5 to ensure Riding Hood can be seen.
  2. Animator
    Controls Riding Hoods animation from Idle, Walking, and Attacking.
  3. Rigidbody 2D
    Unity's built-in physics manager.
  4. Capsule Collider 2D
    Provides a boundary for collision, giving Riding Hood physics.
  5. Character Selection Test
    Testing Next Button, Back Button, and Spamming these Buttons
  6. Character Manager
    Controls the Next Button, Back Button, and Play Button before transitioning to the Level 1 Scene
    Saves the selected Riding Hood for the user.
  7. Player Manager
    Ensures that the Riding Hood picked by the user is properly placed into environment in Level 1 Scene.
    Since Riding Hood is a prefab, the VirtualCinemachine Camera is found and made to target Riding Hood movements.
  8. Player Health
    The activation script when the Player is attacked.
    This activated the HealthBarUI to adjust acordingly.
Setup Instructions:
Drag the Player Manager script into the Player Game Object.
Check that the default prefab, Red, is working with the environment.
Ensure each of the prefabs are the same and work accordingly with the environment

Requirements:
Unity 6000.0.38f1
TextMeshPro
Ink Unity Integration
